# v1.1.0-beta.2 - Options Flow UI Improvements

**Date:** January 2, 2026
**Claude Model:** Opus 4.5 (claude-opus-4-5-20251101)
**Development Tool:** Claude Code (VSCode Extension)

---

## Overview

This beta release improves the Options Flow UI by rearranging fields for better user experience and changing input types for consistency.

## Changes

### Options Flow UI Rearrangement

Rearranged the options flow dialog fields:

| Order | Field | Change |
|-------|-------|--------|
| 1 | Recovery script | Moved up (right after variables description) |
| 2 | Enable repair notifications | Unchanged position |
| 3 | Failures before notification | Changed from slider to input box |
| 4 | Polling Period | Moved to bottom |

### Technical Changes

- **NumberSelector for failures_threshold**: Changed from `vol.Clamp` (renders as slider) to `NumberSelector` with `mode=NumberSelectorMode.BOX` (renders as input box)
- **NumberSelector for scan_interval**: Changed to `NumberSelector` with `unit_of_measurement="seconds"` for better UX
- **Added imports**: `NumberSelector`, `NumberSelectorConfig`, `NumberSelectorMode` from `homeassistant.helpers.selector`

## Files Changed

## Breaking Changes

**None** - Full backward compatibility maintained.

## Upgrade Path

1. Update integration via HACS
2. Restart Home Assistant
3. Options flow will display with new field order
